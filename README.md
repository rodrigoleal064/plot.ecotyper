The \*\*plot.ecotyper\*\* package provides a robust and streamlined R interface for visualizing cell state abundance results generated by \[EcoTyper](https://ecotyper.stanford.edu/). 



It automatically parses the complex EcoTyper output directory structure to generate publication-ready bar plots. The package supports stacked bar plots (absolute abundance), filled bar plots (relative abundance), dynamic sample annotation, and custom faceting.



\## 🚀 Key Features



\* \*\*Smart Parsing:\*\* Automatically reads valid cell type directories (e.g., `B.cells`, `CD8.T.cells`) while safely ignoring metadata folders like `Ecotypes`.

\* \*\*Dual Plot Modes:\*\* Generate a single combined relative abundance plot or individual absolute abundance plots for each cell type.

\* \*\*Dynamic Grouping:\*\* Map substrings in your sample names to clean, presentable groups for automatic `ggplot2` faceting.

\* \*\*Targeted Analysis:\*\* Filter and visualize only specific cell types of interest.

\* \*\*Automated I/O:\*\* Automatically creates output directories if they do not exist.

\## 💻 Usage \& Examples



Load the package to get started:



```r

library(plot.ecotyper)

```



\### 1. Combined Relative Abundance Plot

Generates a single, 100% stacked bar plot showing the proportion of all available cell types across your samples.



```r

plot\_abundance(

&nbsp; folder\_path = "path/to/ecotyper/Recovery",

&nbsp; plot\_type = "combined",

&nbsp; output\_dir = "plots/overview"

)

```



\### 2. Filtering Specific Cell Types

If you are only interested in a specific subset of the microenvironment (e.g., specific lymphoid and myeloid populations), you can explicitly request them using the exact folder names.



```r

plot\_abundance(

&nbsp; folder\_path = "path/to/ecotyper/Recovery",

&nbsp; plot\_type = "combined",

&nbsp; cell\_types = c("B.cells", "CD8.T.cells", "Monocytes.and.Macrophages", "NK.cells"),

&nbsp; output\_dir = "plots/filtered\_lymphoid"

)

```



\### 3. Dynamic Sample Annotation and Faceting

Often, sample IDs contain metadata (e.g., `Patient1\_AMR`, `Patient2\_EUR`). You can provide a named vector to map these substrings to clean group names. The function will automatically group and facet the plots accordingly.



```r

\# Define your dictionary: c("Substring\_to\_match" = "Clean Label")

my\_metadata <- c(

&nbsp; "AMR" = "American",

&nbsp; "EUR" = "European",

&nbsp; "ASN" = "Asian"

)



plot\_abundance(

&nbsp; folder\_path = "path/to/ecotyper/Recovery",

&nbsp; plot\_type = "combined",

&nbsp; sample\_mapping = my\_metadata,

&nbsp; output\_dir = "plots/faceted\_populations"

)

```



\### 4. Absolute Abundance per Cell Type

To inspect the raw abundance (stacking) of states within each specific cell lineage, generate individual plots. The package will save one PNG file per cell type (e.g., `Cell\_type\_abundance\_Fibroblasts.png`).



```r

plot\_abundance(

&nbsp; folder\_path = "path/to/ecotyper/Recovery",

&nbsp; plot\_type = "per\_cell\_type",

&nbsp; sample\_mapping = my\_metadata,  # Faceting works here too!

&nbsp; output\_dir = "plots/individual\_lineages"

)

```

\## 🛠 Function Reference



\### `plot\_abundance()`



| Argument | Type | Default | Description |

| :--- | :--- | :--- | :--- |

| `folder\_path` | `character` | `"files"` | Path to the root EcoTyper output directory containing the cell type subfolders. |

| `plot\_type` | `character` | `"combined"` | `"combined"` (relative abundance of all cells) or `"per\_cell\_type"` (absolute abundance per cell). |

| `filter\_assigned` | `logical` | `TRUE` | If `TRUE`, subsets the data to include only samples present in the `state\_assignment.txt` file. |

| `sample\_mapping`| `named vector`| `NULL` | Maps substrings in sample names to group labels for faceting. E.g., `c("AMR" = "American")`. |

| `cell\_types` | `character` | `NULL` | Specific cell types to plot (must match folder names exactly). If `NULL`, plots all valid folders. |

| `output\_dir` | `character` | `"."` | Path where PNGs will be saved. Created automatically if it does not exist. |
